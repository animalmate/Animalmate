# 00. 에이전트 작업 규칙 (AGENT RULES)

> 이 파일은 AI 코딩 에이전트(Claude Code, Cursor 등)가 **가장 먼저** 읽어야 하는 파일이다.
> 사용 팁: 이 파일을 프로젝트 루트에 `CLAUDE.md` 또는 `.cursorrules`로 복사해두면 자동으로 로드된다.
> 문서 읽는 순서: 00 → 01(기획) → 02(기술스택) → 03(데이터모델) → 04(TODO)

## 프로젝트 한 줄 요약
300명 규모 대학생 봉사 동아리의 운영 자동화 웹서비스. 네이버 카페는 그대로 두고(아카이브),
그 위에 "정기 봉사 공지 → 신청 수합 → 확정 → 카톡 참여코드 배포" 루프와 RAG 챗봇을 얹는다.

## 절대 금지 (MUST NOT) — 위반 시 프로젝트 전체가 무의미해짐
1. **네이버 카페 읽기/크롤링/스크래핑 금지.** 공식 API는 쓰기 전용이다. 로그인 세션 스크래핑,
   Selenium/매크로로 카페 UI 조작은 네이버 약관 위반이며 계정 정지 사유. 절대 구현하지 않는다.
2. **카페 글 "수정/삭제/댓글" API가 있다고 가정하지 말 것.** 존재하지 않는다. 글쓰기(POST)만 있다.
   수정이 필요한 정보는 처음부터 새 서비스 페이지에서 보여주도록 설계한다.
3. **카카오톡 비공식 봇/자동화 금지.** LOCO 프로토콜, 메신저봇류, PC카톡 후킹 전부 약관 위반.
   오픈채팅 생성 API는 제휴사 전용이므로 사용 불가. 카톡방은 사람이 만들고 링크+참여코드만 시스템이 관리한다.
4. **시크릿/개인정보 커밋 금지.** API 키, 토큰, 회원 실명/연락처/학번이 코드·마이그레이션·시드 데이터·
   테스트 픽스처에 절대 들어가면 안 된다. 시드는 반드시 가짜 데이터로.
5. **회원 개인정보(PII)를 RAG 인덱스에 넣지 않는다.** 임베딩 대상 문서에 회원 명단, 연락처,
   계좌번호가 포함되면 안 된다. 문서 저장 파이프라인에 PII 패턴 경고를 넣는다.
6. **권한 검증을 클라이언트에만 두지 않는다.** 모든 쓰기 작업은 서버(API Route/RLS)에서
   역할·소유권·임기를 검증한다. UI에서 버튼을 숨기는 것은 권한 검증이 아니다.
7. **Vercel Cron으로 스케줄 잡을 구현하지 않는다.** Hobby 플랜은 하루 1회 제한에 지정 시각도
   보장되지 않는다. 모든 스케줄은 Supabase pg_cron + pg_net → `/api/cron/*`(CRON_SECRET 검증)
   패턴으로만 구현한다. `vercel.json`에 crons 항목을 추가하지 말 것.

## 필수 원칙 (MUST)
1. **소유권 모델**: 게시물·문서는 `owner_type(personal|team)` + `owner_id`를 가진다.
   수정/삭제 = 소유자(개인 본인 또는 소속 팀원) + 회장단/시스템관리자만.
2. **임기 자동 만료**: 운영진 권한은 `term_end` 경과 시 자동 강등. 크론에서 매일 체크.
3. **RAG visibility 필터**: 문서는 `member|staff|board` 3단계 공개 범위를 가지며,
   챗봇 retrieval 시 질문자 역할 이하의 문서만 검색된다. 이 필터는 검색 쿼리 레벨에서 강제한다.
4. **모든 관리 행위는 audit_logs에 기록** (누가/무엇을/언제/이전값→새값).
5. **외부 API 호출은 실패를 전제로**: 재시도(최대 2회, 지수 백오프) → 실패 시 상태 `failed` +
   운영진 알림. 절대 조용히 삼키지 않는다.
6. **자동 발행 안전장치**: 반복 공지 초안에 필수 필드(일시/장소/정원)가 비어 있으면 발행 보류 +
   담당 팀장단 알림. 빈 공지가 카페에 나가면 안 된다.
7. **챗봇 핸드오프**: 근거 문서가 없거나 확신도가 낮으면 지어내지 말고
   "자료에 없습니다. 운영진(공용 메일)에게 문의해 주세요"로 넘긴다. 답변에는 출처 문서명을 표시한다.
8. **RLS 기본 거부**: 모든 테이블에 RLS를 활성화하고 정책을 부여하지 않는다(= anon key로
   직접 접근 전면 차단). 데이터 접근은 전부 Next.js 서버(service role, 서버 환경변수 전용)를
   경유한다. 브라우저에서 Supabase 테이블을 직접 쿼리하는 코드를 작성하지 말 것.
9. **일시정지 방지 링크 유지**: `/api/health`(경량 DB 조회)와 UptimeRobot 5분 모니터는
   Supabase 무료 티어의 7일 미사용 일시정지를 막는 생명줄이다. 삭제·비활성화 금지.

## 코드 컨벤션
- TypeScript strict 모드. `any` 금지(불가피하면 사유 주석).
- 환경 변수는 `02-TECH-STACK.md`의 목록에만 정의하고 `env.example` 동기화.
- 핵심 로직은 반드시 단위 테스트 작성: 권한 검사, 반복 규칙 날짜 계산(매월 N번째 X요일),
  발행 상태머신 전이, visibility 필터.
- DB 변경은 마이그레이션 파일로만. 스키마를 바꾸면 `03-DATA-MODEL.md`를 같은 커밋에서 갱신.
- 커밋은 작은 단위, 메시지는 한국어 명령형("반복 규칙 날짜 계산 추가").
